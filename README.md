# KERNEL

**Teach Claude Code to remember everything and get smarter with every conversation.**

---

## Why This Matters

- **Claude Code forgets everything.** Close the window, lose your work. KERNEL gives it permanent memory.
- **You want progress, not explanations.** KERNEL makes Claude Code ship working features instead of endless chat.
- **Mistakes should happen once.** KERNEL records what went wrong so Claude never repeats it.

---

## Getting Started

See the complete [setup guide](docs/QUICKSTART.md) for step-by-step instructions.

**Quick version:** Copy the installation block from the setup guide into Claude Code and let it handle everything automatically.

---

## What You Can Do

### Remember Everything
KERNEL gives Claude Code a memory system. Every conversation, every mistake, every solution gets saved. Start a new conversation tomorrow and pick up exactly where you left off.

### Work Bigger
Want to build something that touches lots of moving pieces? KERNEL teaches Claude Code to break big projects into focused tasks and verify the work actually works.

### Learn From Mistakes
Something broke? KERNEL records why so Claude Code never makes that mistake again. Your project gets smarter over time.

### Share Your Work
When you're ready to show someone, KERNEL packages everything up: what changed, why it changed, and proof it works.

---

## Daily Use

**Start working:**
Open Claude Code in your project and just describe what you want. "Add a contact form" or "fix the login bug" - plain English works.

**Check your work:**
Before sharing changes with your team, tell Claude Code to validate everything. It checks the work automatically.

**Save progress:**
Before closing, tell Claude Code to remember where you stopped. Tomorrow starts exactly there.

**When stuck:**
Tell Claude Code what keeps breaking. It saves that forever so the problem never comes back.

---

## When Things Go Wrong

**Claude Code is doing too much:** Say "stop" and be more specific about what you want.

**Something broke:** Your work is saved. You can undo recent changes or go back to any previous version.

**It's confused about what you want:** Restate your request more specifically. Focus on one thing at a time.

**Same mistake keeps happening:** Tell Claude Code to remember the problem permanently. It won't repeat it.

---

## What's Inside

KERNEL adds three things to Claude Code:

1. **Memory** - A `_meta/` folder where Claude Code saves lessons, progress, and what went wrong
2. **Helpers** - Specialized assistants for big tasks (one builds, one checks the work)
3. **Shortcuts** - Quick commands like `/kernel:handoff` (save progress) or "validate my work"

That's it. No configuration needed.
