<variant id="signal">

<mood>Information density. Mission control. Every pixel carries data. The interface of people who know what the numbers mean.</mood>

<sensory>
  <see>Dense grids of small text, perfectly aligned. Blinking status indicators. Multiple data streams simultaneously visible. Color coding that means something. The beauty of a well-organized spreadsheet.</see>
  <feel>Urgency, competence. Being the person who understands the console. The satisfaction of monitoring complex systems. Alert but not anxious.</feel>
  <hear>Radio chatter. Keystroke rhythm. Quiet focus. Occasional alert tone. The hum of systems running nominal.</hear>
</sensory>

<direction>
Maximum information density with clear hierarchy. Monospace type for data, tight spacing, systematic alignment. Status colors (green/amber/red) used functionally, never decoratively—color must mean something. Dense but never cluttered: the difference is hierarchy and alignment. Grid-based to the pixel. The interface rewards expertise—new users see chaos, experts see clarity.
</direction>

<best_for>Admin panels, developer tools, system monitoring, trading interfaces, data-heavy operations, anything where information density is a feature not a bug.</best_for>

<danger>Don't make it unreadable. Signal is dense but organized. If a new user can't find what they need in 3 seconds, you've achieved chaos, not density. Hierarchy is everything.</danger>

</variant>
